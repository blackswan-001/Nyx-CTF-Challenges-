Investigation Framework
How the Attack Works
PHASE 01 — LURE
Phishing Email Delivered
Mass phishing emails spoofing IndusBank are sent to targets. The raw headers contain operator alias, campaign tags, originating IPs, and reply-to leaks — all left behind by the attacker's tooling.
PHASE 02 — KIT
Phishing Kit on Compromised Host
A custom PHP kit clones the bank login page. The kit source recovered from a cPanel server contains developer identity signals: real name, Gmail address, alias, and campaign tag embedded in config files and git residue.
PHASE 03 — HARVEST
Credentials Captured via C2 Panel
847 victim credentials flow into a C2 admin panel. Server-side access logs capture the operator's session IP, which geolocates to a specific city in India — converging with the profile artifact.
PHASE 04 — ATTRIBUTION
Human OSINT — Identity Cross-Match
An archived LinkedIn profile matches the kit developer's Gmail, name, and skills. City, PHP/cPanel expertise, and email are consistent across all four artifacts. Your three-analyst team must triangulate everything to build the attribution flag.
