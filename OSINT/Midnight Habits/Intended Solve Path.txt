----------------------------------------
CRITICAL PASSWORD DETAILS
----------------------------------------

The password MUST be exactly: Blackcoffee!_2021

Validation is case-sensitive and punctuation-sensitive.
Common failures:

  ✗ blackcoffee!_2021      (lowercase B)
  ✗ Black coffee!_2021     (space retained)
  ✗ Blackcoffee_2021       (missing !)
  ✗ BlackCoffee!_2021      (uppercase C)
  ✗ Blackcoffee!2021       (missing underscore)
  ✗ blackcoffee!2021       (multiple errors)

Solvers who fail here have the right idea but
are being imprecise. The friction is intentional —
the challenge teaches that credentials require
exact reconstruction, not approximate guessing.

----------------------------------------
SECURITY IMPLEMENTATION
----------------------------------------

portal.html uses client-side JS validation.
Acceptable for CTF context.

  • User inputs are SHA-256 hashed in-browser
  • Hashes are compared against stored hashes
    (email1, password1) — plaintext never stored
  • Flag is XOR-encrypted with the password as key
  • Flag only decrypts correctly with exact input
  • Stored values in source are hashes + ciphertext
  • DevTools inspection yields nothing actionable

Solvers who attempt to bypass via source inspection
will find only hashes and an encrypted blob.
They must solve the OSINT trail.
